# نظام عاصمة الكون للمصاعد — قاعدة البيانات ووحدة إدارة الطلبات

هذه المرحلة الأولى: العمود الفقري للنظام (قاعدة البيانات + دورة حياة الطلب كاملة). مبنية على **Laravel 11 + PostgreSQL**، وواجهة API تصلح للويب والجوال معاً.

## 1. هرمية البيانات

```
العميل (شخص/شركة)
  └── مبنى (فندق واحة المسك)
        └── مصعد (5 مصاعد Mitsubishi)
              └── طلب (صيانة / عطل / معاينة)
```

طلب الصيانة يرتبط بـ **مصعد محدد** — وهكذا نعرف لاحقاً كم عطلاً تكرّر في كل مصعد.

## 2. الأدوار

| الدور | الصلاحية |
|------|----------|
| admin (المدير) | كل شيء: استقبال الطلبات، تعيين الفنيين، الاطلاع على الكل |
| supervisor (3 مشرفين) | مثل المدير في استقبال الطلبات وتعيين الفنيين |
| technician (فني) | يرى الطلبات المسندة إليه فقط، يبدأ وينهي العمل |
| customer (عميل) | يرفع الطلبات، يتابع طلباته، يقيّم الفني |

## 3. دورة حياة الطلب

```
new (جديد)
  → assigned (عيّنه المدير/المشرف لفني)
    → in_progress (بدأ الفني العمل)
      → completed (أنهى الفني) ──> يُرسل للعميل إشعار تقييم
        → closed (بعد أن يقيّم العميل الفني)

ويمكن الإلغاء (cancelled) من أي حالة مفتوحة.
```

كل انتقال يُسجَّل في `request_status_logs` (من فعله ومتى) — للمساءلة الكاملة. والانتقالات غير المنطقية مرفوضة برمجياً في `RequestService`.

## 4. كيف يلبّي متطلباتك

- **العميل يرفع طلب صيانة/عطل/معاينة:** عبر `POST /api/requests` (الحقل `type`).
- **المدير + 3 مشرفين يستقبلون الطلب:** عند الإنشاء يصلهم إشعار `NewRequestNotification`.
- **أي منهم يعيّن الفني:** `POST /api/requests/{id}/assign` (محمي: المدير والمشرفون فقط).
- **الموظفون ينشئون طلباً نيابة عن العميل:** نفس `POST /api/requests` — يختارون مصعد العميل.
- **التقييم بعد الإنهاء:** عند `complete` يصل العميل إشعار، ويقيّم عبر `POST /api/requests/{id}/rate`، فيُغلق الطلب.

## 5. التشغيل

```bash
composer create-project laravel/laravel elevator   # أو ضع الملفات في مشروعك
composer require laravel/sanctum                    # للمصادقة (ويب + جوال)

# انسخ ملفات هذا المجلد إلى مشروعك (database/, app/, routes/)
# اضبط اتصال PostgreSQL في ملف .env ثم:
php artisan migrate
```

## 6. ملاحظات للجوال (Flutter/React Native)

- نفس مسارات API تخدم الجوال. استخدم Sanctum tokens للمصادقة.
- للإشعارات الفورية: أضف قناة `fcm` في ملفات `Notifications` (موضّحة بتعليق `// أضِف 'fcm'`).

## 7. الخطوات التالية المقترحة

1. وحدة العقود والفواتير والمدفوعات (مع بوابة دفع).
2. وحدة المخزون وقطع الغيار (مع خصم تلقائي عند الاستخدام في الطلب).
3. لوحة المؤشرات والتقارير (أعطال متكررة لكل مصعد، أداء كل فني من متوسط التقييمات).
4. جدولة الصيانة الدورية تلقائياً حسب العقد.

## بنية الملفات

```
database/migrations/   → جداول قاعدة البيانات (7 ملفات)
app/Models/            → النماذج والعلاقات
app/Services/          → RequestService (منطق دورة الحياة)
app/Http/Controllers/  → المتحكمات (الطلبات + التقييم)
app/Notifications/     → الإشعارات (جديد / تعيين / إنهاء)
routes/api.php         → مسارات API
docs/database-erd.mermaid → مخطط العلاقات
```

## ترتيب تشغيل الهجرات

الهجرات مرقّمة بالتسلسل (000001 → 000012) لتُنفَّذ بالترتيب الصحيح للعلاقات. شغّل `php artisan migrate` بعد ضبط اتصال PostgreSQL.
